10.¿Qué son los métodos / verbos de petición HTTP? Dar ejemplos. Bibliografía MDN

los métodos de petición indican la acción que se desea realizar para un recurso determinado.
Aunque estos también pueden ser sustantivos, estos métodos de solicitud a veces son llamados HTTP verbs.
GET

GET: El método GET solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos.
HEAD: El método HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
POST: El método POST se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
PUT: El modo PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
DELETE: El método DELETE borra un recurso en específico.
CONNECT: El método CONNECT establece un túnel hacia el servidor identificado por el recurso.
OPTIONS: El método OPTIONS es utilizado para describir las opciones de comunicación para el recurso de destino.
TRACE: El método TRACE realiza una prueba de bucle de retorno de mensaje a lo largo de la ruta al recurso de destino.
PATCH: El método PATCH es utilizado para aplicar modificaciones parciales a un recurso.

11. ¿Qué son los códigos de estado de respuestas HTTP? Realizar una tabla con su clasificación. Bibliografía MDN

Los códigos de estado de respuesta HTTP indican si se ha completado satisfactoriamente una solicitud HTTP específica.

Tabla de codigos de estdo de respuestas HTTP.

Clasificacion codigo nombre descripcion
Respuestas informativas(100–199) 100 Continue Indica que todo esta bien y debe continuar.
101 Switching Protocol Se envía en respuesta a un encabezado de solicitud.
102 Processing indica que el servidor ha recibido la solicitud y aún se encuentra procesandola.
103 Early Hints Permite que el agente de usuario empiece a pre-cargar recursos mientras el servidor prepara una respuesta.
Respuestas satisfactorias (200–299) 200 OK La solicitud ha tenido éxito.
201 Created la respuesta enviada después de una petición PUT.
202 Accepted La solicitud se ha recibido, pero aún no se ha actuado.
203 Non-Authoritative Information La petición se ha completado con éxito, pero su contenido no se ha obtenido de la fuente originalmente solicitada.
204 No Content La petición se ha completado con éxito pero su respuesta no tiene ningún contenido (Sin cache).
205 Reset Content La petición se ha completado con éxito pero su respuesta no tiene ningún contenido (Cache).
206 Partial Content La petición servirá parcialmente el contenido solicitado.
207 Multi-Status transmite información sobre varios recursos
208 Already Reported El listado de elementos DAV ya se notificó previamente, por lo que no se van a volver a listar.
